# 献立アプリ MVP要件定義

## 概要

ふたり暮らし向け週間献立管理Webアプリ。
ホットクック中心の自炊生活を効率化し、現在のGeminiチャット運用をアプリに置き換える。

## ユーザー

| 名前 | 役割 |
|------|------|
| れん | 主に料理・買い出し・献立計画担当 |
| あかね | レシピ確認・調理担当 |

## 運用フロー

1. **週1回**：残り物 + 週の予定（1人/2人、昼/夜）をAIに伝える → 献立提案
2. **献立確定** → 買い物リスト自動生成
3. **買い出し** → リストにチェック（2人でリアルタイム同期）
4. **当日** → レシピ確認して調理

## MVP機能（4つ）

### F1. 週間献立（カレンダー形式）

- 月〜日の週間表示（週切り替え可能）
- 各日に昼・夜の2枠（`meal_type`: `"lunch"` | `"dinner"` のみ。breakfastなし）
- 各枠に人数（1人 or 2人）を設定
- 献立カードタップ → レシピ詳細へ遷移
- 長押し → 編集（外食に変更、レシピ差し替え、人数変更）
- 空スロット「+」→ AI提案 or 手動登録
- 存在しない週を表示した場合は空状態を返す（自動作成しない）

### F2. 買い物リスト

- 献立確定時に、全レシピの材料を集約して自動生成
- 食材集約ルール：同一食材名 + 同一単位のみ合算。異なる単位の場合は別行表示
- チェックボックス付き（is_checked, checked_by）
- Supabase Realtimeで2人のチェック状態をリアルタイム同期
- 手動で食材を追加・削除可能

### F3. レシピ表示

- 献立タップ → レシピ詳細画面
- タイトル、材料一覧（人数に応じた分量計算）、手順
- ホットクック情報：メニュー番号、まぜ技ユニット有無
- 下ごしらえ時間・加熱時間の表示
- AI不介在（DBから直接取得）

### F4. AI献立提案

- チャット形式UI
- ユーザーが残り物と予定を入力 → AIが1週間の献立を提案
- ホットクック前提のレシピ優先
- 納得いくまでやり取り → 「確定」で保存
- 確定 → 買い物リスト自動生成
- 直近2週間の献立をコンテキストに含めてマンネリ防止
- Gemini 2.5 Flash（Function Calling対応）

## 技術スタック（家計簿アプリと統一）

| 項目 | 技術 |
|------|------|
| フレームワーク | Next.js 15 (App Router) |
| 言語 | TypeScript 5.7 (strict mode) |
| UI | React 19 |
| DB | Supabase（家計簿と同じプロジェクト、DB共有） |
| AI | Gemini 2.5 Flash（Function Calling） |
| ホスティング | Vercel |
| スタイル | Tailwind CSS + Radix UI |
| 認証 | Supabase Auth（家計簿と共有、Google OAuth） |

## 後回し機能（Phase 2以降）

- バーチャル冷蔵庫（在庫自動管理）
- 評価・殿堂入りシステム
- 家計簿連携（食費自動記録、予算連動）
- クッキングモードUI（スワイプ式手順表示）
- 売り場別リスト自動分類

## 家計簿アプリとの関係

- 同じSupabaseプロジェクト内にテーブルを追加
- 認証・ユーザー管理は共有
- MVPではFKを張らない（将来Phase 2で `shopping_lists` と `transactions` を連携）
- `user_type`（共同/れん/あかね）の運用ルールを踏襲
